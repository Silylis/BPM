合同管理需求说明文档

 1. 项目概述

 1.1 背景

基于运输管理系统，需要建立完善的合同管理模块，用于管理与客户签署的运输服务合同。合同管理作为业务管理的核心模块，需要与货物管理、运输计划、报价管理等模块深度集成，支持复杂的多车多DN业务场景。

 1.2 目标

- 建立完整的合同全生命周期管理体系

- 支持合同与客户/供应商的关联管理

 1.3 适用范围

适用于运输管理系统中所有运输服务合同的管理，包括：

- BMW与物流服务商的框架协议

- 供应商与物流服务商的运输合同

- 专项运输服务合同

- 紧急运输服务合同

 2. 功能需求

 2.1 合同列表管理

 2.1.1 列表字段展示

- 合同编号: 系统唯一标识，支持搜索和排序

- 合同名称: 合同标准名称，支持模糊搜索

- 合同类型: 显示合同分类，支持筛选

- 客户名称: 客户方名称，支持搜索

- 会计单位名称: 服务方名称，支持搜索

- 签订日期: 合同签署日期，支持日期范围筛选

- 合同开始时间: 合同生效开始时间

- 合同结束时间: 合同到期时间

- 状态: 合同当前状态，支持状态筛选

  - 草稿（灰色标识）

- 创建人: 合同创建人员信息

- 创建时间: 合同创建时间

- 操作: 操作按钮组

  - 查看（所有状态可用）

  - 编辑（草稿状态可用）

  - 删除（草稿状态可用）

 2.1.2 查询和筛选功能

- 快速搜索: 支持合同编号、合同名称、客户名称、供应商名称的组合搜索

- 高级筛选:

  - 合同类型多选筛选

  - 合同状态多选筛选

  - 签订日期范围筛选

  - 创建人筛选

- 排序功能: 支持按创建时间、签订日期、到期时间排序

- 分页显示: 支持每页10/20/50/100条记录选择

 2.1.3 批量操作功能

- 批量删除: 批量删除草稿状态的合同

- 批量导出: 导出选中合同的基础信息到Excel

- 批量审批: 批量审批待审批状态的合同

 2.2 新增/编辑合同管理

 2.2.1 基础信息组（必填信息）

（1）合同编号:

  - 系统自动生成，格式：CNT-YYYYMMDD-XXX

  - 支持手动修改（需验证唯一性）

  - 字段类型：文本框，最大长度50字符

  - 必填项，不能为空

（2）合同名称:

  - 合同标准名称，建议格式：客户名称+运输服务合同

  - 字段类型：文本框，最大长度200字符

  - 必填项，不能为空

（3）合同类型:

  - 字段类型：下拉选择框

  - 必填项，必须选择

  - 选项：

    - 框架协议

    - 项目合同 

    - 单次运输

（4）合同开始日期:

  - 字段类型：日期选择器

  - 必填项，不能为空

  - 默认值：当前日期

  - 不能晚于合同结束日期

  - 合同的生效日期

（5）合同结束日期:

  - 字段类型：日期选择器

  - 必填项，不能为空

  - 不能早于开始日期

  - 合同的到期日

 2.2.2 签约主体组（必填信息）

（1） 客户名称:

  - 字段类型：下拉选择框（可搜索）

  - 必填项，必须选择

  - 数据源：客户管理系统

  - 选择后自动带出联系人和电话

（2）客户联系人:

  - 字段类型：文本框，最大长度50字符

  - 必填项，不能为空

  - 可手动修改自动带出的信息

（3）客户联系电话:

  - 字段类型：文本框，手机号码格式验证

  - 必填项，不能为空

  - 格式验证：11位手机号码或带区号的固定电话

（4）会计单位名称:

  - 字段类型：下拉选择框

  - 必填项，必须选择

  - 数据源：供应商管理系统

  - 选择后自动带出联系人和电话

（5）会计单位联系人:

  - 字段类型：文本框，最大长度50字符

  - 必填项，不能为空

  - 可手动修改自动带出的信息

（6）会计单位联系电话:

  - 字段类型：文本框，手机号码格式验证

  - 必填项，不能为空

  - 格式验证：11位手机号码或带区号的固定电话

 2.2.3 财务信息组（部分必填）

（1）支付方式:

  - 字段类型：下拉选择框（可多选）

  - 选项：

    - 网银支付

    - 线下支付

    - 承兑汇票 

（2）支付币种:

  - 字段类型：下拉选择项（可多选）

  - 选填项

（3）合同金额:

  - 字段类型：数字输入框，支持小数点后2位

  - 选填项，框架协议可不填具体金额

  - 单位：元（人民币）

  - 最小值：0，最大值：999,999,999,999.99

 2.2.4 其他信息组（选填信息）

（1）回单类型:

  - 字段类型：下拉选择项（可多选）

  - 选填项

（2）其他备注:

  - 字段类型：文本框

  - 选填项，200字内

（3）合同附件:

  - 字段类型：文件上传组件

  - 选填项，支持多文件上传

  - 支持格式：PDF、DOC、DOCX、JPG、PNG

  - 单文件最大：10MB

  - 总文件数：最多10个

 2.2.5 表单验证规则

- 实时验证: 失去焦点时立即验证字段格式和内容

- 提交验证: 点击保存/提交时进行全表单验证

- 错误提示: 错误字段红色边框 + 具体错误信息提示

- 成功提示: 保存成功后显示成功消息并跳转到列表页

 2.3 合同状态管理

 2.3.1 状态定义和说明

- 草稿:

  - 初始状态，允许所有字段编辑

  - 可以删除

  - 不参与业务流程

  - 界面显示：灰色标识

 2.3.2 状态流转规则

- 草稿 → 待审批:

  - 触发条件：点击"提交审批"按钮

  - 前置条件：所有必填字段已填写且格式正确

  - 后置动作：发送审批通知给相关人员

- 待审批 → 已生效:

  - 触发条件：审批通过且达到生效日期

  - 前置条件：审批流程完成且审批结果为通过

  - 后置动作：合同正式生效，可用于业务流程

- 待审批 → 草稿:

  - 触发条件：审批驳回

  - 前置条件：审批人员驳回申请

  - 后置动作：发送驳回通知，允许重新编辑

- 已生效 → 即将到期:

  - 触发条件：系统自动检查（每日定时任务）

  - 前置条件：距离结束时间 <= 30天

  - 后置动作：发送到期提醒通知

- 已生效/即将到期 → 已到期:

  - 触发条件：系统自动检查（每日定时任务）

  - 前置条件：当前日期 > 结束时间

  - 后置动作：停止使用该合同进行新业务

- 已生效/即将到期 → 已终止:

  - 触发条件：点击"终止合同"按钮

  - 前置条件：有终止合同权限的用户操作

  - 后置动作：立即停止使用该合同

 2.3.3 删除规则

- 可删除状态: 仅草稿状态的合同可以删除

- 删除权限: 需要合同删除权限或合同创建人

- 删除确认: 需要二次确认，防止误删除

- 级联检查: 删除前检查是否有关联的报价单、运输计划等

- 软删除: 建议使用软删除，保留数据记录但标记为已删除
