 1. 项目背景


 1.1 业务背景

车线管理是运输管理系统的核心功能模块，用于管理运输路线的基础信息，包括起点终点、距离时长、中转点等关键信息。通过规范化的车线管理，可以提高运输计划的准确性和执行效率。


 1.2 现状分析

- 需要统一管理所有运输路线信息

- 支持多起点多终点的复杂路线配置

- 需要灵活的中转点设置功能

- 要求便捷的路线查询和筛选能力


 2. 功能需求


 2.1 车线列表管理


 2.1.1 列表展示功能

数据展示字段：

- 序号：列表序号，便于快速定位

- 车线编码：唯一标识码（如：SH-BJ-001、SH-SZ-002）

- 车线名称：路线名称（如：上海-北京专线、上海-深圳专线）

- 起点：起始地点（如：上海浦东新区）

- 终点：目的地点（如：北京市朝阳区）

- 中转点：中转站点数量显示

- 距离(公里)：路线总距离

- 时长(小时)：预计运输时间

- 状态：启用/停用状态

- 录入人：创建人员信息

- 最后时间：最后修改时间

- 操作：编辑、复制、删除等操作按钮


 2.1.2 查询筛选功能

搜索条件：

- 车线编码搜索：支持模糊匹配车线编码

- 车线名称搜索：支持模糊匹配车线名称

- 起点筛选：下拉选择起点地区

- 终点筛选：下拉选择终点地区

- 状态筛选：启用/停用状态筛选


操作功能：

- 新增车线：创建新的运输路线

- 导入：批量导入车线数据

- 导出：导出车线列表数据


 2.2 新增车线功能


 2.2.1 基础信息配置

必填字段：

- 车线编码：手动输入，格式如：SH-BJ-001，具备唯一性

- 车线名称：描述性名称，如：上海-北京专线


 2.2.2 起终点配置

起点配置：

- 省份/地区：选择起点省份（如：上海），地址库选取，必填

- 城市：选择起点城市（如：上海），，地址库选取，必填

- 区县：选择具体区县（如：浦东新区），，地址库选取，必填

- 详细地址：输入具体地址信息


终点配置：

- 省份/地区：选择终点省份（如：北京），地址库选取，必填

- 城市：选择终点城市（如：北京），地址库选取，必填

- 区县：选择具体区县（如：朝阳区），地址库选取，必填

- 详细地址：输入具体地址信息


地图选点：调用高德地图选址（优先级可以滞后）


 2.2.3 中转点配置

中转点管理：

- 添加中转点：支持添加多个中转站点

- 中转点信息：

  - 省份/地区选择

  - 城市选择

  - 区县选择

  - 详细地址输入

- 中转点排序：支持拖拽调整中转点顺序

- 删除中转点：支持删除不需要的中转点


批量操作：

- 清空中转点：一键清除所有中转点

- 模板应用：应用常用中转点模板


 2.2.4 路线信息配置

基础信息：

- 预计里程(公里)：输入路线总距离

- 预计时长(小时)：输入预计运输时间

- 备注说明：路线相关备注信息


 2.2.5 表单操作

- 取消：取消新增操作，返回列表页

- 保存：保存车线信息并返回列表页


 2.3 车线编辑功能


 2.3.1 编辑入口

- 在车线列表的操作列中点击"编辑"按钮

- 弹出编辑弹窗，预填充现有数据


 2.3.2 编辑功能

- 支持修改所有车线信息字段

- 保持与新增功能相同的表单结构

- 支持中转点的增删改操作


2.5 车线删除功能


 2.5.1 删除确认

- 点击删除按钮弹出确认对话框

- 显示删除警告信息

- 确认后执行删除操作


 2.5.2 删除限制

- 检查车线是否被运输计划引用

- 如有关联数据，禁止删除并提示

- 支持强制删除选项（管理员权限）


 3. 业务规则


 3.1 数据验证规则


 3.1.1 必填字段验证

- 车线编码：必填，唯一性校验

- 车线名称：必填，长度限制100字符

- 起点信息：省份、城市必选

- 终点信息：省份、城市必选


 3.1.2 格式验证

- 车线编码：支持字母数字和连字符，如SH-BJ-001

- 距离：正数，支持小数点后2位

- 时长：正数，支持小数点后2位


 3.1.3 中转点规则

- 最多支持5个中转点

- 中转点不能与起点终点重复

- 支持拖拽调整顺序
