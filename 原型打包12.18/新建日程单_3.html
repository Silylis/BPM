 新增日程单功能需求文档

 1. 概述

 1.1 文档目的

本文档旨在明确新增日程单功能的产品需求，为设计、开发提供详细的功能说明和交互规范。

 1.2 功能简介

新增日程单功能允许用户创建销售日程单，配置基础信息、线路货物信息以及其他信息，支持价格单自动关联和换绑功能。

 2. 功能需求

 2.1 基础信息配置

2.1.2 日程单号

- 日程单号：灰色背景的只读字段，显示自动生成的编号--当前需要用户手动维护

 2.1.3 基本字段配置

第一行字段：

- 供应商（必填）：下拉选择框，支持供应商选择

- 供应商合同（必填）：下拉选择框，基于供应商联动显示合同

- 供应商联系人（必填）：下拉选择框，基于供应商联动显示联系人 

- 账期类型-参考供应商合同

第二行字段：

- 生效日期（必填）：日期选择器，带日历图标

- 截止失效日：日期选择器，带日历图标

- 纳税级别（必填）：文本输入框，税率，数字，两位小数

- 支付天数（必填）：数字输入框，右侧显示"天"

 2.1.4 字段验证规则

- 供应商、供应商合同、供应商联系人、生效日期、访问代码、支付天数为必填项

- 截止失效日期如果填写，必须晚于生效日期

- 支付天数必须为正整数

 2.1.5 字段联动逻辑

- 选择供应商后，供应商合同下拉框自动加载该供应商的合同选项

- 选择供应商后，供应商联系人下拉框自动加载该供应商的联系人选项

- 供应商变更时，相关联的合同和联系人字段需要重置

 2.2 线路&货物信息配置

 2.2.1 动态行配置

支持多行线路&货物信息录入，每行包含以下字段：

- 线路名称：下拉选择框（如：北京-上海）

- 线路别名：下拉选择框（如：特快）

以上两个字段互相联动

- 货物品类：下拉选择框

- 货物名称：下拉选择框

- 车型：下拉选择

- 车长：下拉选择

- 是否往返：下拉是/否

 2.2.2 行操作功能

- +添加：蓝色按钮，点击可新增一行线路&货物信息

- 删除：红色按钮，点击可删除当前行（至少保留一行）

- 支持拖拽排序

 2.2.3 验证规则

- 以上均为单位，以上7个字段组成唯一条件，新增时判断是否存在和相同的组合项，存在不允许新增

 2.3 其他信息配置

 2.4 价格单关联配置（重要功能）

 2.4.1 智能关联机制

默认关联规则：

- 销售日程单默认与日程单内供应商ID一致的价格单绑定

- 系统根据供应商合同自动查找对应价格单

- 支持价格单的自动创建和手动换绑

 2.4.2 换绑功能

- 用户可以取消自动关联，手动选择其他价格单

- 提供价格单搜索功能，支持按编码、名称检索

- 换绑操作需要确认提示，避免误操作

- 记录换绑历史，便于追溯

 2.6 操作按钮

 2.6.1 主要操作

- 保存：蓝色按钮，保存当前日程单信息

- 保存并关闭：灰色按钮，保存后关闭当前页面，保存后，默认启用状态

- 保存并新建价格单：蓝色按钮，保存日程单并跳转到新建价格单页面

 2.6.2 操作逻辑

- 保存前执行完整的表单验证

- 保存成功后给予用户明确的反馈提示

- 支持快捷键操作（Ctrl+S保存）

 3. 交互行为

 3.1 表单验证

- 实时验证必填字段，未填写时显示红色星号

- 数据格式验证（日期、数字等）

- 业务逻辑验证（如日期先后关系）

- 验证失败时高亮显示错误字段并提示错误信息

 3.2 用户体验

- 字段获得焦点时高亮显示

- 下拉框支持键盘输入快速检索

- 日期选择器提供友好的日历界面

- 页面响应式设计，适配不同屏幕尺寸

 3.3 数据保存

- 支持自动草稿保存功能

- 页面离开前提示未保存的更改

- 保存失败时保留用户输入的数据

 4. 业务规则

 4.1 价格单关联规则

1. 默认关联：销售日程单自动关联与供应商ID一致的价格单

2. 智能匹配：系统优先匹配供应商合同对应的价格单

3. 自动创建：如无匹配价格单且开启自动创建，系统将新建价格单

4. 可换绑性：用户可随时修改价格单关联关系

5. 历史记录：系统记录所有价格单关联变更历史

 4.2 数据完整性规则

1. 必填字段校验：供应商、供应商合同、供应商联系人、生效日期、访问代码、支付天数为必填项

2. 数据格式校验：确保各字段数据符合预定格式

3. 业务逻辑校验：如日期逻辑、数值范围等

4. 关联数据校验：确保关联的基础数据存在且有效

 4.3 权限控制规则

1. 不同角色用户可操作的字段范围不同

2. 某些关键字段可能需要特殊权限才能修改

3. 价格单关联操作可能需要审批权限
